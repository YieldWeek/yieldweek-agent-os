# Edition Playbook Workflow

## Purpose

Define, document, and improve the YieldWeek edition format so it can be repeated across cities and eventually franchised.

## Trigger

A new city edition is planned, an edition closes, or the edition format needs updating.

## Agents involved

- Primary agent: Product Agent
- Supporting agents: Operations Agent, Editorial Agent, Commercial Agent, Badge Certification Agent, Franchisee Manager Agent, Orchestrator Agent

## Inputs

- City edition plan
- Audience profile
- Sponsor requirements
- Content/agenda structure
- Operations plan
- Post-event report
- Attendee/sponsor feedback

## Steps

1. Define edition standard
   - Owner: Product Agent
   - Output: edition format baseline

2. Align operations
   - Owner: Operations Agent
   - Output: execution feasibility notes

3. Align editorial structure
   - Owner: Editorial Agent
   - Output: agenda and content structure fit

4. Align commercial constraints
   - Owner: Commercial Agent
   - Output: sponsor deliverable and access-tier constraints

5. Update badge quality criteria
   - Owner: Badge Certification Agent
   - Output: edition-level criteria for third-party event certification

6. Version playbook
   - Owner: Product Agent
   - Output: internal playbook version

7. Franchise readiness check
   - Owner: Franchisee Manager Agent
   - Output: ready/not-ready signal and gaps

8. Founder review
   - Owner: Orchestrator Agent
   - Output: approved or revised playbook

## Human approval points

Founder approval required before format standard changes, ticket/access tier changes, sponsor-impacting format changes, or franchise-ready designation.

## Outputs

- Edition format standard
- Attendee experience map
- Quality criteria
- Playbook version notes
- Franchise readiness status

## Success metrics

- Singapore format documented before launch
- post-edition playbook update completed within 30 days
- format can be repeated in next city
- franchise readiness based on documented standard, not improvisation
