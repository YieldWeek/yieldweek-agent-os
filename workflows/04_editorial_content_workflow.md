# Editorial Content Workflow

## Purpose

Produce category-building content that supports YieldWeek credibility, sponsor conversion, speaker interest, and city edition gravity.

## Trigger

A content topic, city theme, sponsor narrative, or market development needs to be turned into publishable content.

## Agents involved

- Primary agent: Editorial Agent
- Supporting agents: Market Intelligence Agent, Reality Checker Agent, Orchestrator Agent

## Inputs

- Topic or thesis
- Target audience
- Channel
- Desired output format
- Required sources
- Approval level

## Steps

1. Content brief
   - Owner: Orchestrator Agent
   - Output: topic, audience, channel, purpose, deadline

2. Source gathering
   - Owner: Market Intelligence Agent
   - Output: source pack with links and key facts

3. Draft
   - Owner: Editorial Agent
   - Output: first draft

4. Claim review
   - Owner: Reality Checker Agent
   - Output: approved, edited, or blocked draft

5. Final approval
   - Owner: Orchestrator Agent
   - Output: publish, revise, or escalate

## Human approval points

Founder approval required before publishing public content that makes strategic, market-size, partnership, sponsor, speaker, or competitive claims.

## Outputs

- Content brief
- Source pack
- Draft
- Reality check notes
- Final publish-ready version

## Success metrics

- supports sponsor or organiser conversations
- improves category clarity
- contains verified claims
- creates reusable narrative assets
