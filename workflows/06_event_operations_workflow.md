# Event Operations Workflow

## Purpose

Execute a YieldWeek city edition with clear ownership, operational discipline, and sponsor-quality reporting.

## Trigger

A YieldWeek owned, co-produced, or badged event moves from planning to execution.

## Agents involved

- Primary agent: Operations Agent
- Supporting agents: Commercial Agent, Editorial Agent, Reality Checker Agent, Orchestrator Agent

## Inputs

- Event title
- City and venue
- Date and time
- Format
- Sponsor commitments
- Speaker commitments
- Registration page
- Run-of-show requirements

## Steps

1. Operating brief
   - Owner: Operations Agent
   - Output: event operating brief

2. Sponsor deliverables check
   - Owner: Commercial Agent
   - Output: sponsor activation checklist

3. Editorial/programming check
   - Owner: Editorial Agent
   - Output: agenda and speaker status

4. Risk and claims review
   - Owner: Reality Checker Agent
   - Output: reviewed event page, sponsor language, and public claims

5. Execution plan
   - Owner: Operations Agent
   - Output: run-of-show and responsibility matrix

6. Founder review
   - Owner: Orchestrator Agent
   - Output: approve, revise, or escalate

7. Post-event report
   - Owner: Operations Agent
   - Output: event recap, issues, attendee metrics, sponsor notes

## Human approval points

Founder approval required for venue commitments, spend, sponsor deliverables, public event pages, final agenda, and any change that affects brand or economics.

## Outputs

- Event operating brief
- Run-of-show
- Sponsor activation checklist
- Public page review
- Post-event report

## Success metrics

- event executed without major operational failure
- sponsor deliverables fulfilled
- attendee experience preserved
- usable post-event data captured
