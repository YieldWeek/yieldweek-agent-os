# Media Distribution Workflow

## Purpose

Distribute approved YieldWeek content through the right channels and track whether distribution supports category credibility, sponsor interest, organiser interest, and speaker interest.

## Trigger

A piece of content, newsletter issue, press release, or media partner update is approved for distribution.

## Agents involved

- Primary agent: Media / Distribution Agent
- Supporting agents: Editorial Agent, Commercial Agent, Legal / Compliance Agent, Reality Checker Agent, Orchestrator Agent

## Inputs

- Approved content asset
- Target channel
- Target audience
- Distribution date
- Required approvals
- Media partner constraints
- Sponsor or commercial relevance

## Steps

1. Confirm content readiness
   - Owner: Editorial Agent
   - Output: approved content draft

2. Verify claims and legal exposure
   - Owner: Reality Checker Agent and Legal / Compliance Agent when needed
   - Output: approved distribution copy or escalation

3. Build distribution plan
   - Owner: Media / Distribution Agent
   - Output: channel plan, timing, and copy variants

4. Align commercial moments
   - Owner: Commercial Agent
   - Output: sponsor/organiser outreach windows tied to content

5. Founder approval
   - Owner: Orchestrator Agent
   - Output: publish, revise, or block

6. Publish or schedule
   - Owner: Media / Distribution Agent
   - Output: published/scheduled record

7. Track performance
   - Owner: Media / Distribution Agent
   - Output: engagement and inbound signal summary

## Human approval points

Founder approval required for press releases, milestone announcements, partnership claims, named quotes, paid distribution, and any content posted from a named individual account.

## Outputs

- Distribution plan
- Channel copy
- Approval status
- Published links
- Performance notes
- Commercial follow-up triggers

## Success metrics

- relevant inbound generated
- sponsor/organiser conversations supported
- no unsupported public claims
- media partner status tracked
