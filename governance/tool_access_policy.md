# Tool Access Policy

This policy defines what agents may access and where human approval is required.

## Default principle

Agents should use the minimum tool access required to complete the assigned workflow. Access to email, CRM, payments, website publishing, attendee data, and production systems must be constrained.

## Access matrix

| Agent | Allowed tools | Restricted tools | Approval notes |
|---|---|---|---|
| Orchestrator | CRM read/write for tasks and routing, document drafting | Payments, contract signature, publication | May coordinate but not approve restricted actions |
| Market Intelligence | Web research, CRM create/enrich for accounts/contacts/events | Email sending to prospects | Must mark source confidence |
| Badge Certification | Event scoring, badge memo drafting, event status recommendation | Final badge approval | Founder approval required for certification/listing |
| Commercial | Approved outbound templates, CRM opportunity/interactions updates | Pricing changes, contract commitment, refunds | Founder approval for non-standard terms |
| Editorial | Drafting, research, speaker/content briefs | Public publication | Reality Checker and publication approval required |
| Operations | Luma prep, vendor checklist, run-of-show drafting | Vendor commitment, attendee data export | Founder approval for spend and data sharing |
| Reality Checker | Claim review, risk logging | Commercial approval, legal advice | Escalate high/critical risk |
| Finance/Treasury | Invoice drafts, reconciliation log, payment status tracking | Refunds, discounts, tax/legal advice | Founder approval for any money movement commitment |
| Legal/Compliance | Contract review memo, privacy checklist, risk log | Legal representation without counsel/founder | External counsel needed when legal opinion required |
| Media/Distribution | Approved content scheduling, media log | Unapproved publication, exclusivity | Founder approval for public commitments |
| Product | Edition playbook, format standards | Commercial promises | Founder approval for franchise readiness |
| Tech/Engineer | Website draft changes, incident logs, data architecture memos | Production publish without approval, attendee data sharing | Approval required before registration/data collection |
| Franchisee Manager | Franchisee monitoring and fit memos | Franchise sales before readiness gate | Founder approval always required |

## Prohibited tool behaviors

- Sending external email without logging interaction.
- Publishing public content without approval trace.
- Exporting or sharing attendee data without approved purpose.
- Changing website production content without approval.
- Accessing payment systems beyond approved finance workflow.
- Making commitments in Discord, email, CRM, or docs that exceed agent authority.
